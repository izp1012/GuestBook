4.5 목록 처리

목록 처리시 고려해야하는 작업
1. 화면에서 필요한 목록 데이터에 대한 DTO 생성
-> 목록 처리는 거의모든 게시판 관련 기능에서 사용될 가능성이 높기 떄문에 재사용이 가능한 구조로 생성하기!
-> 모든 목록을 처리하는 기능에는 페이지 번호나 한 페이지 당 몇개나 출력될 것인가 와 같은 공통 부분이 많기때문에 이를 클래스를 이용하자

2. DTO 를 Pageable 타입으로 전환 (페이지 요청 처리 DTO)
작성하려고하는 PageRequestDTO 는 목록 페이지를 요청할 때 사용하는 데이터를 잿용하기 쉽게 만드는 클래스
목록 화면에서는 페이지 처리를 하는 경우가 많이 있기 때문에
'페이지 번호', '페이지 내 목록의 갯수' 등을 검색조건으로 많이 사용
PageRequestDTO 는 이러한 파라미터를 DTO 로 선언하고 나중에 재사용하는 용도로 많이 사용

화면에서 전달되는 목록 관련된 데이터에 대한 DTO 를 PageRequestDTO 라는 이름으로 생성하고,
화면에서 필요한 결과는 PageResultDTO 라는 이름의 클래스로 생성한다

3. Page<Entity> 를 화면에서 사용하기 쉬운 DTO 의 리스트 등으로 변환
- PageRequestDTO, PageResultDTO 에서 각각 처리

4. 화면에 필요한 페이지 번호 처리
- 서비스 계층 에서는 PageRequestDTO 를 파라미터로 PageResultDTO 를 리턴 타입으로 사용하는 getList() 를 설계하고,
엔티티 객체를 DTO 객체로 변환하는 entityToDto() 를 정의한다.
